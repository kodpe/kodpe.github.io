Linux, WSL et shells

Usages en enseignement, apprentissage et monde professionnel

1. Linux et WSL sont-ils plus utilisés ?

Dans l’enseignement supérieur et les métiers techniques, Linux (et WSL) est aujourd’hui plus utilisé que Windows seul.

Cela ne signifie pas que Windows disparaît, mais que les environnements techniques reposent majoritairement sur Linux.

Windows reste dominant pour le grand public, Linux domine pour les serveurs, le développement et l’ingénierie.

2. Pourquoi Linux est privilégié dans les écoles

Un standard historique

Les universités utilisaient Unix bien avant Linux. Linux a hérité de cette culture : terminal, fichiers, processus, scripts.

Gratuit et accessible

  • pas de licence
  • installable sur n’importe quel ordinateur
  • identique pour tous les étudiants

Pédagogiquement efficace

Linux oblige à comprendre le fonctionnement réel d’un système :

  • processus
  • permissions
  • réseau
  • automatisation

3. Le rôle clé de WSL

WSL (Windows Subsystem for Linux) permet d’utiliser Linux directement à l’intérieur de Windows.

Il est très utilisé dans l’enseignement car :

  • beaucoup d’étudiants ont Windows
  • WSL évite l’installation complexe de Linux
  • les commandes sont exactement les mêmes que sur un serveur
ls
cd
pwd
WSL est aujourd’hui le compromis pédagogique le plus courant.

4. Linux dans le monde professionnel

Serveurs et cloud

La quasi-totalité des serveurs professionnels tournent sous Linux :

  • serveurs web
  • bases de données
  • cloud
  • containers (Docker, Kubernetes)

Développement et ingénierie

Linux est omniprésent en :

  • développement logiciel
  • data science
  • intelligence artificielle
  • cybersécurité
  • DevOps

5. Quel shell apprendre en priorité ?

La question du shell est centrale dans l’apprentissage du terminal.

Le choix recommandé

Bash est le shell à apprendre en priorité.

Pourquoi Bash ?

  • shell par défaut sur la majorité des systèmes Linux
  • présent sur les serveurs
  • utilisé dans WSL
  • standard de la documentation et des scripts
#!/bin/bash
ls | grep ".txt"

6. Et les autres shells ?

Zsh

Plus ergonomique, très populaire chez les développeurs, mais basé sur Bash.

→ Excellent après Bash

Fish

Très convivial, mais incompatible avec Bash.

→ Mauvais choix pour débuter dans un cadre académique

PowerShell

Shell Windows puissant, orienté objets.

→ Utile uniquement pour des contextes Windows spécifiques

7. Recommandation pédagogique finale

  • Utiliser Linux ou WSL pour l’apprentissage
  • Apprendre le terminal via Bash
  • Se concentrer sur les concepts Unix
  • Découvrir d’autres shells ensuite
Ce choix garantit une continuité entre l’école, les stages et le monde professionnel.
